CSS Variables

Introduction

Custom Properties are also known as CSS variables or Cascading Variables. They are user-defined variables that are declared once and can be reused in your stylesheet. It is widely supported on nearly all browsers with over 95% compatibility rate. It is Used by declaring a custom property on the :root pseudo class (globally) or within a css class (locally). Some of the common uses of css custom properties are mentioned below:

  • Box shadows
  • Gradients
  • Color themes (light / dark mode)
  • Grids
  • Font scaling
  • Responsive layout (set responsive themes for text and font size on smaller screens)
:root{ -- color:#ffd100; }

What does it do?

CSS custom properties allows you to create reusable values for things such as colours, font sizes, or spacing. The value is declared by using two dashes followed by the property name, which will allow you to be able to reference it throughout the CSS document.

Once a CSS custom property is defined, it can be referenced by using var() instead of using a standard property value.

Since a custom property inherits its value from their parent element, you can use the @property rule to control its inheritance. This also allows you define what kind of value it accepts, its default value, and whether it should inherit its value from its parent element.

Common Issues

Not scoping them correctly

If you declare a variable in one selector, other selectors won’t be able to access it unless they are descendants (nested within it).

Solutions:

  • Check that your variable is properly scoped.
  • Declare the variable globally using the :root pseudo-selector if you want to access it throughout your CSS file.

Not adding a fallback value

In situations where a value is invalid (e.g., --bg-color: 16px), or you forget to define a variable altogether, a fallback value ensures a substitution can be used. If you don’t add a fallback, then the variable would default to its initial or inherited value.

Solutions:

  • Add a fallback value to your variables, or multiple fallback values (in case the first one isn’t applied for whatever reason).
  • Review your code to ensure you are defining variables and using valid values.

Overusing custom properties

While CSS variables offer more flexibility, overusing them can complicate things by creating cluttered code that is difficult to manage.

Solutions:

  • Avoid using them for every single thing and minor layout changes.
  • Limit usage of variables to values that will be reused multiple times or that are likely to change.
